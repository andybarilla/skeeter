Implement a `github.Store` that satisfies the `store.Store` interface using the
GitHub Contents API. This enables the Wails desktop app (and any remote client)
to read and write tasks without a local git clone.

Use the GitHub REST API (`repos/{owner}/{repo}/contents/{path}`) for file CRUD.
Auth should support the `gh` CLI token first (`gh auth token`), falling back to
a `GITHUB_TOKEN` env var.

## Acceptance Criteria

- [ ] `github.Store` implements `store.Store` (List, Get, Create, Update, NextID)
- [ ] Reads/writes task files via GitHub Contents API
- [ ] Handles SHA-based conflict detection (required for PUT/DELETE on Contents API)
- [ ] Auth via `gh auth token` output, falling back to `GITHUB_TOKEN` env var
- [ ] CLI gains `--remote owner/repo` flag to use the GitHub backend instead of local filesystem
- [ ] Config is read from the remote `.skeeter/config.yaml`
- [ ] Works with both public and private repositories

## Context

- `internal/store/store.go` — the `Store` interface this must implement
- `internal/store/filesystem.go` — reference implementation to match behavior
- GitHub Contents API: `GET/PUT /repos/{owner}/{repo}/contents/{path}`
- Consider a `internal/store/github.go` file alongside the filesystem impl
